# AdReel

One product photo + one hook line → an ~8-second vertical (9:16) video ad with native spoken audio and music, generated by Google Veo.

The spoken first-3-seconds hook is the conversion lever — the sound is the product.

## Stack

- Next.js 15 (App Router), TypeScript strict, Tailwind CSS v4
- `@google/genai` with `veo-3.1-fast-generate-preview` on the Gemini API
- Stateless — no DB. The Veo operation name is carried client-side between poll calls.

## How it works

Veo generation takes minutes; a serverless function can't stay open that long. The flow is kick-off + client-side poll:

1. `POST /api/generate` — starts the Veo generation, returns `{ operationName }` in seconds.
2. `GET /api/status?op=…` — the client polls every 8s; one REST status check per call.
3. `GET /api/video?uri=…` — proxies the authenticated Veo file URI and streams the mp4, so the API key stays server-side. Only Google's generative-language host is proxied.

## Run it

```bash
npm install
cp .env.example .env.local   # then put your real key in .env.local
npm run dev
```

Get a key at https://aistudio.google.com/apikey — Veo requires billing enabled on the key's project.

`GEMINI_API_KEY` is read only inside `/api/*` routes. It is never exposed client-side and never committed.

## Deploy

Vercel: set `GEMINI_API_KEY` in project env vars **before** the production deploy.

The app is host-portable — no Vercel-only APIs in core logic. It also deploys to Cloud Run with `gcloud run deploy --source .`.
